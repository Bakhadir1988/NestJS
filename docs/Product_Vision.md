# Kanban API — Product Vision

## Краткое описание

Kanban API — это backend-приложение для управления проектами и задачами по методологии Kanban. Проект ориентирован на чистую архитектуру, масштабируемость и production-подход.

Проект создаётся как учебный, но максимально приближенный к реальному коммерческому продукту.

Референс продукта: **Trello API, Asana API**.

---

## Целевая аудитория

- Индивидуальные пользователи и небольшие команды для управления задачами.
- Мобильные и web-клиенты, которым нужен backend для Kanban-доски.

---

## Проблема, которую решает продукт

- Сложность интеграции управления задачами в другие приложения из-за отсутствия простых, API-first решений.
- Необходимость в гибком и расширяемом backend для визуализации рабочих процессов.

---

## Цели продукта

- Позволить пользователю:
  - создавать и управлять досками (boards).
  - добавлять и настраивать колонки (columns) на досках.
  - создавать, редактировать и перемещать задачи (tasks) между колонками.
- Обеспечить:
  - безопасную аутентификацию и авторизацию.
  - строгую валидацию данных на всех уровнях.
  - расширяемую и понятную модульную архитектуру.

---

## Нефункциональные требования

- Чёткая модульная архитектура (NestJS).
- Типобезопасность (TypeScript).
- Валидация на уровне входных данных (DTO, Pipes).
- Подготовка к масштабированию.
- Готовность к production (логирование, конфигурация).

---

## Границы MVP

В MVP **НЕ входят**:

- Real-time обновления (WebSockets).
- Сложные системы прав (администратор доски, гость, участник).
- Прикрепление файлов к задачам.
- Комментарии к задачам.
- Полнотекстовый поиск.

Эти функции могут быть добавлены на следующих этапах.
